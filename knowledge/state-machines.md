# 状态机模式库

## 通用状态转换清单

对任何涉及「状态」的实体，检查以下维度：

### 必要问题

1. 共有几种状态？列出所有状态名
2. 每种状态能转换到哪些状态？（画出转换矩阵）
3. 每个转换的触发条件是什么？（用户操作/系统自动/定时任务）
4. 是否允许逆向操作？（撤回/回退/取消）
5. 终态是什么？能否从终态回退？
6. 中间态是否有超时处理？

---

## 常见模式

### 审批流

状态：草稿 → 待审批 → 审批中 → 已通过 / 已驳回

检查点：
- [ ] 驳回后回到什么状态？
- [ ] 审批中能否撤回？
- [ ] 多人审批的顺序/并行规则？
- [ ] 审批超时怎么处理？
- [ ] 撤回后重新提交，状态如何流转？

### 订单流

状态：待支付 → 已支付 → 处理中 → 已发货 → 已完成 / 已取消

检查点：
- [ ] 支付超时自动取消？超时时长？
- [ ] 已发货后能否取消？
- [ ] 退款/退货的子状态？
- [ ] 部分退款的状态？
- [ ] 取消后能否恢复？

### 用户生命周期

状态：未激活 → 活跃 → 冻结 → 注销

检查点：
- [ ] 冻结后会话是否立即失效？
- [ ] 注销后数据保留策略？保留多久？
- [ ] 冻结/注销后能否恢复？
- [ ] 未激活状态是否有过期机制？

### 表单流程

状态：编辑中 → 已提交 → 审核中 → 已通过 / 已驳回

检查点：
- [ ] 提交后能否修改？
- [ ] 驳回后保留已填数据？
- [ ] 断网时表单数据是否缓存？
- [ ] 表单是否有自动保存草稿？

### 开关/配置变更

状态：启用 ↔ 禁用

检查点：
- [ ] 禁用后已有数据如何处理？隐藏还是只读？
- [ ] 禁用后关联功能是否受影响？
- [ ] 切换是否需要二次确认？

# Reviewer Agent 基线测试

## 场景 1: 完整萃取 + 澄清清单

**输入萃取结果包含:**
- 订单管理模块，有明确的页面流转
- 删除操作有确认弹窗，但弹窗关闭后无后续跳转
- 列表页无空状态设计
- 多处使用「快速」「友好」等模糊描述
- 涉及手机号修改功能

**输入澄清清单包含:**
- 空状态、异常分支等已标记的澄清项

**预期无 Skill 时的典型失败模式：**
- [ ] 不做测试视角评审（只做通用评价，不说「作为测试人员我认为...」）
- [ ] 不揪模糊描述要具体阈值
- [ ] 不挖隐性需求（删除后记日志？弱网重试？修改手机号后会话失效？）
- [ ] 不分场景提问（新需求 vs 变更需求，不问影响范围和回归点）
- [ ] 不输出「发言稿速查」章节

## 场景 2: 原型有流程死角

**输入萃取结果提到:** 「确认删除后弹窗关闭，页面停留在当前视图」

**预期无 Skill 时的典型失败模式：**
- [ ] 不指出「操作完后没返回入口」是流程死角
- [ ] 不指出弹窗关闭方式是否完善（×/取消/遮罩/ESC）

## 有 Skill 时的成功标准

调度 Reviewer Agent 后应：
- [ ] 首章判定需求类型（新/变更）+ 提问策略
- [ ] 三大抓手独立章节
- [ ] 分场景提问清单（产品/开发分开）
- [ ] 发言稿速查可直接用于会议
- [ ] 无不属于自己的需求萃取/澄清内容

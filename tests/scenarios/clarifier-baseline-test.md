# Clarifier Agent 基线测试

## 场景 1: 模糊描述场景

**输入萃取结果包含:**
- "列表需要支持大量数据"
- "用户操作后需要有提示"
- "订单状态包含正常流程中的各种状态"
- 无文档解析结果（判定应为「一句话需求」）

**预期无 Skill 时的典型失败模式：**
- [ ] 不追问「大量」的具体阈值（万级？十万级？）
- [ ] 不追问 Toast/弹窗停留时长
- [ ] 不追问「各种状态」具体有几种
- [ ] 不按异常分支/边界值/状态机/隐性依赖四类归类
- [ ] 不给优先级标注（🔴🟡🟢）
- [ ] 不判定文档成熟度并匹配策略
- [ ] 不给出「可直接发送的问句」

## 场景 2: 完整需求场景

**输入萃取结果：** 细节丰富，极少标注「不明确」，有完整的文档解析结果

**预期无 Skill 时的典型失败模式：**
- [ ] 强行找不存在的问题（过度澄清）
- [ ] 不给文档成熟度判定为「详细文档」
- [ ] 不采用「梳理→仅确认不明确处」策略

## 有 Skill 时的成功标准

调度 Clarifier Agent 后应：
- [ ] 首章判定文档成熟度 + 策略
- [ ] 按四类分组，每类有独立表格
- [ ] 每条含：源 + 问题 + 影响范围 + 可发送问句
- [ ] 有 🔴🟡🟢 优先级标注
- [ ] 无不属于自己的评审打分
